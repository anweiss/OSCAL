Control: control assembly

A structured information object representing a security or privacy control. Each security or privacy control within the Catalog is defined by a distinct control instance.

The control assembly has the following contents (in order):

  • id flag (required)
  • class flag (optional)
  • title field (one)
  • param assemblies (zero or more)
  • prop fields (zero or more)
  • link fields (zero or more)
  • part assemblies (zero or more)
  • subcontrol assemblies (zero or more)
  • references assembly (zero or one)

Controls may be grouped using group, and controls may be partitioned using part or enhanced using subcontrol.
